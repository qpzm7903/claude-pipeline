# 任务: 功能开发 (feature-dev)

## 任务目标

根据 `plan.md` 中规划的功能需求，完成新功能的设计、实现和测试。

---

## 任务执行流程

### 1. 需求分析

- 阅读 `plan.md` 和 `issues.md`，确定本轮要开发的功能
- 理解功能的业务背景和技术约束
- **每轮只做一个功能点**，保持小步快跑

### 2. 方案设计

在动手编码之前，先输出设计决策：

```
## 决策: <功能名称>实现方案
- 现状: <当前代码中相关模块的状态>
- 方案: <实现思路，涉及哪些模块/类/接口>
- 影响范围: <哪些现有代码会受影响>
- 风险: <可能的风险点>
```

### 3. 实现

- 按照方案设计编写代码
- 遵循项目现有的代码风格和架构约定
- 在修改的地方添加准确简单易懂的简体中文日志
- **最小化改动**：只改必须改的代码

### 4. 测试

- 为新功能编写单元测试
- 运行全量测试确保没有引入回归
- 如果没有现有测试框架，先搭建

### 5. 代码自审

提交前对 `git diff` 进行自审，检查：
- 是否有遗留的调试代码
- 错误处理是否完整
- 是否符合项目架构约定
- 是否有硬编码的魔法值

---

## 注意事项

- 涉及新模块/新依赖时，在 ADR 中记录决策（参考文档治理规则）
- 新 API 必须补充 Javadoc 注释
- 复杂业务逻辑必须补充行内注释
- 不要过度设计：先让功能可用，后续迭代优化

# job-demo · 功能开发任务

## 项目上下文

仓库：https://github.com/qpzm7903/job-demo
当前状态：已完成 v0.5.x（OAuth2 授权服务器骨架，含 client_credentials / authorization_code / JWK 端点 / 单元测试）。

技术栈（**硬约束，不可改**）：
- JDK 17 / Maven（禁 Gradle）
- Spring Boot 3.2.x
- Spring Authorization Server 1.x
- H2 内存数据库（仅验证用）
- JUnit 5

## 本任务目标

在现有 v0.5.x 上**扩展功能**，每轮交付一个**最小可用的功能点**，遵循小步快跑：

### 候选功能（按 plan.md 优先级选 1 个）

1. **客户端管理 API**（v0.6.0）
   - `POST /admin/clients` 注册新 OAuth2 客户端（client_id / client_secret / scopes）
   - `GET /admin/clients` 列出已注册客户端
   - `DELETE /admin/clients/{id}` 删除客户端
   - 持久化到 H2（用 JPA 即可，启动时建表）
   - 鉴权：本管理端点需用 `client_credentials` token 访问

2. **Token 自省端点**（v0.7.0）
   - 实现标准 RFC 7662 `POST /oauth2/introspect`
   - 输入 token，返回 `{active, sub, scope, exp, ...}`

3. **Refresh Token 流**（v0.8.0）
   - 在 authorization_code 流中签发 refresh_token
   - 实现 grant_type=refresh_token 的换发逻辑

读 `plan.md`，挑下一个**未完成**的版本作为本轮目标；每轮只做一个版本。

## 每轮严格流程

1. **Plan**：阅读 plan.md → 选定本轮版本号与功能点 → 输出决策块（任务模板已要求）
2. **Build baseline**：`mvn clean test > /tmp/baseline.log 2>&1; echo "EXIT=$?" >> /tmp/baseline.log; tail -10 /tmp/baseline.log`
   - EXIT≠0 → 本轮目标改为"修复构建/测试"，用 `fix(...)` 提交
3. **Develop**：实现新功能，**只动本版本范围内的文件**
4. **Test**：为新功能写至少 2 个测试（happy path + error path）；`mvn test > /tmp/after.log 2>&1; tail -20 /tmp/after.log` 必须 EXIT=0
5. **Commit & Push & Tag**：
   - Conventional Commits 简体中文：`feat(clients): 新增客户端管理 API`
   - `git tag v0.X.0 && git push origin main --tags`
   - 本轮**必须产生 ≥ 1 commit**

## 输出控制（防 token 爆炸）

- Maven 输出一律重定向到 `/tmp/*.log` 后 tail，禁止把完整输出带入对话
- 禁止 `cat target/`、`find` 大目录

## Commit 规范（红线）

- `feat(<范围>): <描述>` 简体中文
- **严禁** AI 署名（`Co-Authored-By: Claude` / `Written by AI` / `Generated by`）
- 不附 `Signed-off-by`
- git 用户：默认（已配置为 qpzm7903）

## 禁止事项

- 禁止改 v0.5.x 已签发的 API 路径与响应结构（向后兼容）
- 禁止引入 Lombok 之外的代码生成依赖
- 禁止把 H2 改为外部 DB（本任务范围内仍用内存 H2）
